Лабораторна робота 3. Взаємодія додатків засобами черг повідомлень.

Завдання: Розробити пару Publisher-Subscriber, які взаємодіють через чергу. Можна розвинути додаток із першої або другої лабораторної, додавши на певну подію Вашого сервера відправку повідомлення (у форматі JSON або XML) в чергу. Це буде Producer Також необхідно розробити Consumer, який отримуватиме повідомлення із черги і певним чином реагуватиме на них.

Також варто розглянути різні аспекти конфігурування exchange, binding і поведінгу черги в разі якщо обробник повідомлення вибросив виключення (Exception)

Опис проекту

Ця лабораторна робота реалізує REST API для системи відстеження фітнес-активностей з використанням Spring Boot та RabbitMQ для асинхронної обробки подій.

Основні цілі проекту:

Розробка REST API для управління користувачами, вправами та тренуваннями
Інтеграція RabbitMQ для обробки подій у реальному часі
Реалізація патерну Publisher-Subscriber
Обробка помилок через Dead Letter Queue (DLQ)
